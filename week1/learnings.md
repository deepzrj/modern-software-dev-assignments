# Week 1 – Prompt Engineering

## Techniques Implemented

1. K-Shot Prompting  
Used examples to guide model behavior for deterministic string transformation.

2. Chain-of-Thought  
Guided reasoning using structured step-by-step logic for math problems.

3. Tool Calling  
Forced model to delegate tasks via structured JSON tool calls.

4. Self-Consistency  
Improved answer reliability by leveraging multiple reasoning paths.

5. RAG (Retrieval-Augmented Generation)  
Ensured answers were grounded in external API documentation.

6. Reflexion  
Enabled self-correction by prompting the model to review and refine outputs.

## Key Learnings

- Small models require strict prompts  
- Structure > vague instructions  
- Examples strongly influence outputs  
- Grounding prevents hallucination  
- Iteration improves reliability
